# GitHub & Deployment Guide

## Step 1 — Install Git (if you haven't)

Download from https://git-scm.com/download/win  
Run installer with all defaults.

Verify:
```cmd
git --version
```

---

## Step 2 — Configure Git (once ever)

```cmd
git config --global user.name "Your Name"
git config --global user.email "your@email.com"
```

---

## Step 3 — Create GitHub repo

1. Go to **https://github.com/new**
2. Repository name: `lensid`
3. Set to **Public** (needed for free deployment later)
4. Do NOT check "Add README" — you already have one
5. Click **Create repository**

GitHub shows you two commands. Ignore them — use the ones below instead.

---

## Step 4 — Push your code

```cmd
cd C:\Users\ivans\Downloads\lensid\lensid

git init
git add .
git commit -m "initial: LensID material intelligence platform"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/lensid.git
git push -u origin main
```

Replace `YOUR_USERNAME` with your actual GitHub username.

After this, every time you make changes:
```cmd
git add .
git commit -m "describe what you changed"
git push
```

---

## Step 5 — What goes in .gitignore (already set up)

The `.gitignore` already excludes:
- `node_modules/` — npm reinstalls these
- `.venv/` — pip reinstalls these  
- `backend/data/*.pt` — model weights (too large, share separately)
- `backend/data/*.csv` — generated by setup_dataset.py
- `backend/data/images/` — add this line if images are large

To add images exclusion:
```cmd
echo backend/data/images/ >> .gitignore
git add .gitignore
git commit -m "exclude dataset images from git"
git push
```

## Deployment — Railway (backend) + Vercel (frontend)

### Backend on Railway

1. Go to **https://railway.app** → sign in with GitHub
2. Click **New Project** → **Deploy from GitHub repo** → select `lensid`
3. Railway auto-detects the Dockerfile in `/backend`
4. Set **Root directory** to `backend`
5. Add environment variable: `PORT=8000`
6. Click Deploy
7. Copy the generated URL e.g. `https://lensid-backend.railway.app`

### Frontend on Vercel

1. Go to **https://vercel.com** → sign in with GitHub
2. Click **Add New Project** → import `lensid`
3. Set **Root directory** to `frontend`
4. Add environment variable:
   ```
   VITE_API_URL=https://lensid-backend.railway.app
   ```
5. Click Deploy

Update `frontend/src/utils/api.ts` to use the env var:
```ts
const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL
    ? `${import.meta.env.VITE_API_URL}/api`
    : '/api'
})
```

After deploy, your app is live at `https://lensid.vercel.app` (or similar).

Every `git push` to `main` auto-deploys both services.

## Important: model weights for deployment

The weights file (`clip_lr4e4_gelu_rf2_r1_best.pt`) is ~9MB — fine for GitHub.

```cmd
# Copy weights into the repo's data folder
copy "C:\Users\ivans\Downloads\material_fingerprint\3_models\model_C-MLP\clip_lr4e4_gelu_rf2_r1_best.pt" "backend\data\"

# Update .gitignore to NOT exclude .pt files if you want them in git
# Or use Railway's volume/environment to provide the file separately
```

Simplest for a demo: include the weights in git (9MB is fine).

```cmd
# Remove the *.pt exclusion from .gitignore, then:
git add backend/data/clip_lr4e4_gelu_rf2_r1_best.pt
git commit -m "add model weights"
git push
```

Railway will then have the weights and CLIP will run in production.
